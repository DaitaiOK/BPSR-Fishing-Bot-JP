BPSR Fishing Bot（日本語改変版）

本プロジェクトは、Python + 画像認識（OpenCV） を用いて
ゲーム内の釣りを自動化する オープンソースのフィッシングボット です。

海外の GPL-3.0 ライセンス のオープンソースプロジェクトをベースに、
実運用での 安定性・実用性を重視 して仕様整理・改変を行っています。

概要

本ボットは 有限状態機械（FSM / Finite State Machine） を採用し、
釣りに必要な一連の操作を ステート遷移 によって自動実行します。

各ステートの遷移は主に 画面の画像認識結果 をトリガーとして行われ、
画像認識に失敗した場合でも タイムアウト処理によって安全に復帰 する
フェイルセーフ設計となっています。

動作条件（重要）

以下の条件を 必ず満たしてください。

Windows 環境

Python 3.8 以上

ゲームは フルスクリーンモード

解像度は 1920 × 1080

ゲーム画面を 前面表示

キャラクターが
「釣り場でキャスト可能な状態」 にいること

本プログラムは 必ず管理者権限で起動

※ 管理者権限がない場合、
マウス・キーボード操作が正常に行われません。

※ 上記条件を満たさない場合、
画像認識精度の低下や誤動作が発生する可能性があります。

起動方法

ゲームを起動し、釣り場でキャスト可能な状態にする

本プログラムを 管理者権限で起動

ゲーム画面を前面にした状態で、以下のキー操作を行います

ホットキー操作
キー	機能
7キー	ボットを起動
8キー	一時停止 / 再開
9キー	ROI（画像認識の検出範囲）を画面上に表示
0キー	完全停止（終了処理・入力解放）

※ ROI表示（9キー） はデバッグ・調整用途です。

ステート構成と処理フロー

本ボットは以下の順でステート遷移を行います。

竿チェック
   ↓
キャスト
   ↓
アタリ待ち
   ↓
ミニゲーム
   ↓
リザルト処理
   ↓
竿チェック（最初に戻る）

各ステートの役割
① 竿チェック（Checking Rod State）

使用中の釣り竿が正常かを画像認識で確認

破損している場合は自動で交換

② キャスト（Casting Bait State）

キャスト可能な状態を確認後、釣りを開始

③ アタリ待ち（Waiting for Bite State）

画面上の変化を監視し、アタリを検知

④ ミニゲーム（Playing Minigame State）

魚とのやり取りを自動操作

状況に応じて左右操作を実行

⑤ リザルト処理（Finishing State）

成功・失敗を判定

次の釣りへ移行

フェイルセーフ仕様

各ステートには タイムアウト が設定されています

画像認識に失敗した場合でも、
一定時間経過後に必ず 「竿チェック」ステートへ復帰

無限停止・フリーズを起こしにくい構成です

画像認識について

OpenCV の テンプレートマッチング を使用

テンプレート画像は以下に配置されています

src/fishbot/assets/templates/


ゲーム側の UI 変更などで認識しなくなった場合は、
該当箇所の画像を撮り直して差し替えてください。

設定ファイルについて

設定ファイルは以下にまとめられています

src/fishbot/config/


通常利用では ユーザーが設定を変更する必要はありません。
動作調整が必要な場合のみ編集してください。
